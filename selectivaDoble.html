Ahora o más tarde...

Ahora ejecutemos el siguiente código en PSeInt. Hagámoslo una vez dando la respuesta SI, y otra dando la repuesta NO.

Pseudocódigo

PROCESO Nombre_1

DEFINIR nombre, varAmigo COMO CARACTER

ESCRIBIR "Hola, ¿cómo te llamas?"

LEER nombre

ESCRIBIR "Hola, ", nombre, " un gusto conocerte"

ESCRIBIR "Quieres agregar amigos a tus contactos? (SI/NO)"

LEER varAmigo 

SI varAmigo = "SI" ENTONCES

ESCRIBIR "Genial !! Vamos a sumar amigos !!"

SINO 

ESCRIBIR "Bien, luego podrás hacerlo de todas maneras"

FINSI

FINPROCESO

¿Qué sucedió? Si como respuesta contestamos SI, entonces la salida del algoritmo fue, "Genial !! Vamos a sumar amigos !!" Mientras que si la respuesta fue NO, la salida fue "Bien, luego podrás hacerlo de todas maneras". Es decir, no hubo una única secuencia de acciones, sino más de una alternativa.

Hemos visto que la salida impresa cambia dependiendo de nuestra respuesta. Esto difiere del caso del saludo en la sección previa, pues allí siempre la secuencia de acciones es la misma. Aquí, existen acciones posibles que dependen de si previamente se verificó el cumplimiento de una condición o no. Esta estructura se conoce como Estructura Selectiva o Condicional. En este caso, se trata de una selectiva doble. También existen otros tipos de selectivas, como la simple o la múltiple.

El inicio de una selectiva se da con la evaluación de una condición. Una pregunta que sólo puede tener dos respuestas válidas: verdadero o falso. Para lograr esto, en la condición siempre comparamos una variable con un valor. Cuando ejecutemos el algoritmo, la variable tendrá un valor dado, el cual se comparará con el otro valor que planteamos (en gral., lo vamos a presentar como una constante numérica o un texto). En nuestro caso, si respondemos "SI", estamos ingresando ese valor a la variable varAmigo, y luego esa variable se compara con el texto "SI": la condición queda SI = SI, lo cual se evalua, evidentemente, como verdadero.
