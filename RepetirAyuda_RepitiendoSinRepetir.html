Repitiendo sin repetir...

Bien, aprovechando que la lógica de algoritmos nos lo permite, ahora replanteamos nuestra solución para saludar a nuestro grupo de tres amigos, de la siguiente manera::

Pseudocódigo

PROCESO Nombre_3

DEFINIR contVueltas COMO ENTERO

DEFINIR nombre COMO CARACTER

PARA contVueltas <- 1 HASTA 3 CON PASO 1 HACER

ESCRIBIR " ************************* "

ESCRIBIR "Hola, ¿cómo te llamas?"

LEER nombre

ESCRIBIR "Hola ", nombre, ", un gusto conocerte" 

FINPARA

ESCRIBIR "Es un gusto conocerlos a todos :-) "

FINPROCESO

Observando...

Observen que la cantidad de nombres que nos pide ingresar el algoritmo es tres. Entonces son tres los ciclos o iteraciones que realiza (uno por cada amigo). Luego de ingresar el tercero, el algoritmo ya no tiene más elementos para procesar, y entonces sale del bucle. Este tipo de estructura en el algoritmo que nos permite repetir una y otra vez una serie de instrucciones, siempre que conozcamos la cantidad de repeticiones que necesitamos realizar, se conoce como Repetitiva Para (o Desde).

Juguemos en PSeInt...

Utilizando la función “Ejecución Paso a Paso” de PSeInt, veamos qué es lo que va sucediendo en la consola de interfaz:
1. ¿En qué línea el algoritmo se detiene esperando el ingreso de un dato?
2. ¿Qué impresiones va realizando?
3. ¿A dónde regresa el algoritmo una vez que repite instrucciones?; es decir, una vez que realiza una nueva iteración.
4. ¿Qué instrucciones vuelve a ejecutar?
